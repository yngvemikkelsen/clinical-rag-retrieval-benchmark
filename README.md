# clinical-rag-retrieval-benchmark
clinical-rag-retrieval-benchmark
Replication materials for: "Embedding Model Selection for Clinical Retrieval-Augmented Generation: A Factorial Benchmark Study"
Submitted to JMIR Medical Informatics.
Preprint: none
Overview
This repository contains all code, data, and prompts needed to reproduce the experiments reported in the paper. The study benchmarks 10 embedding models and BM25 across three clinical corpora, two query formats, and four chunking strategies, totaling 294 experimental conditions. A factorial ANOVA with η² effect sizes quantifies the relative contributions of embedding model choice versus clinical context variables (corpus type, query format) to retrieval performance.

Reproducing the Experiments
Main benchmark
bashpython scripts/clinical_rag_benchmark_v3.py
This script:

Downloads and preprocesses MTSamples and PMC-Patients
Generates 500 synthetic clinical notes using Mistral-7B-Instruct-v0.2 (or loads from cache)
Builds keyword and natural language queries for each corpus
Evaluates all embedding models and BM25 across chunking strategies
Runs factorial ANOVA and bootstrap confidence intervals
Outputs results to data/all_results.csv

Estimated runtime: 2–3 hours on a single GPU.
Validation experiment
bashpython scripts/metadata_query_validation.py
Generates reduced-lexical-dependence queries from GPT-4o-extracted metadata and evaluates retrieval stability.
Metrics
Primary: MRR@10, P@1, Recall@10/20/50/100, NDCG@10
All metrics use a known-item retrieval design (one correct document per query). Bootstrap 95% confidence intervals are computed with 10,000 resamples.
